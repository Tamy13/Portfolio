Pythonで機械学習実践¶

scikit-learnのLightGBMを使ってKaggleのコンペティション「Predict Future Sales」と言う1ヶ月後の売上数予測を実践します。
また、Kagglerでも意外にやっている人が少ない「コーディングを綺麗に書く」ことも意識して作っています。 コーディングを綺麗に書くことで以下のメリットがあります。

・可読性の向上
・どこのセルを何回実行しようともバグもエラーもでない
・過去に実行したデータがどういった構造だったのか復習がしやすい
・保守性の向上(コードの書き換えが楽になる)

実際のデータ分析でコーディングが汚いと上記の恩恵を受けれず、後で無駄な時間を消耗してしまいがちなのでコーディングは綺麗に書くことをオススメします。

日単位 → 月単位¶

現在のデータは日単位となっているので、月単位での処理ができるように データを月で丸めていきます。

In [4]:
# 時系列データを月で丸める。

# item_priceは平均で、item_cnt_dayは合計丸める
df_group_item = df_1.groupby(["date_block_num", "shop_id", "item_id"], as_index=False)["item_price", "item_category_id", "item_cnt_day"]\
                    .agg({"item_price":"mean", "item_category_id":"mean", "item_cnt_day":"sum"})\
                    .rename(columns={'item_cnt_day': 'item_cnt_month'})  #目的変数:item_cnt_month
 
# dateは月に丸めてから年と月の特徴量を作成する
# (以下の処理をまとめて実行)
# ・データを必要な特徴量のみに絞る
# ・dateを日付型に変換
# ・dateをindexにセットする
# ・月単位でグループ化
# ・date_block_numの値が変わらないように平均でとる
# ・indexからdateを取り出す
# ・年の特徴量と月の特徴量作成(数値型)
# ・dateを削除
df_years_months = df_1.filter(["date","date_block_num"])\
                      .assign(date = lambda x:pd.to_datetime(x["date"],  format="%d.%m.%Y"))\
                      .set_index('date')\
                      .resample('M')\
                      .mean()\
                      .reset_index()\
                      .assign(
                          years = lambda x:[date.year for date in x["date"]]    # 年の特徴量作成
                        , months = lambda x:[date.month for date in x["date"]]  # 月の特徴量作成
                        )\
                      .drop(columns=["date"])  # 日付型はのちの学習に使えないのでここで削除

#作成した特徴量を1つに結合
df_2 = pd.merge(df_group_item, df_years_months, on = ["date_block_num"])\
         .assign(date_block_num = lambda x:x["date_block_num"] + 1)\
         .reindex(columns=["years", "months", "date_block_num", "shop_id", "item_id", "item_price", "item_category_id", "item_cnt_month"])

del df_group_item, df_years_months

ここでは様々な組み合わせでグループ化をしたり、基本統計量を取得したり、掛け合わせをしたりすることで多様な特徴量を作成します。

# 特徴量作成

# カテゴリーごとの平均の値段、売上数、売上の3つを作成
df_category = df_2.groupby(["date_block_num","shop_id","item_category_id"], as_index=False)["item_price", "item_cnt_month"]\
                  .agg({"item_price":"mean", "item_cnt_month":"sum"})\
                  .rename(columns = {"item_price": "category_price", "item_cnt_month": "category_cnt_month"})\
                  .assign(category_sales = lambda x:x["category_price"] * x["category_cnt_month"])

# お店の商品の売上
df_shop_item_sales = df_2.filter(["date_block_num", "shop_id", "item_id", "item_price" ,"item_cnt_month"])\
                         .assign(shop_item_sales = lambda x:x["item_price"] * x["item_cnt_month"])\
                         .drop(columns = ["item_price", "item_cnt_month"])

# お店ごとの売上合計
df_shop_sales = df_shop_item_sales.groupby(["date_block_num", "shop_id"], as_index=False)["shop_item_sales"]\
                                  .sum()\
                                  .rename(columns={"shop_item_sales": "shop_sales"})

# 商品の平均額

# 作成した特徴量を1つに結合

ラグ特徴量作成¶

次に、ラグ特徴量を作成していきます。
ラグ特徴量は時系列データにおいて過去の情報を持つ特徴量のことを言います。
通常はshift関数で行の並び順をズラすことでラグ特徴量を作成します。
しかし、今回はデータが日付だけでなくshop_id、item_idとも主キーになっているので、単純に行の並び順をズラして作成はできません。
そこで、1月ずつデータを区切り、ラグ特徴量を作成し、最後に区切ったデータを元のデータと結合する方法でラグ特徴量を作成します。

In [6]:
# ラグ特徴量作成

# ラグ特徴量作成用関数
def func_lag(x, j): # x:特徴量, j:ラグの期間
  tmp_before = df_3.query("date_block_num ==  @i - @j")\
                   .filter(["shop_id", "item_id", x])\
                   .rename(columns = {x: x + "_" + str(j)})  # query関数は条件文でデータを簡単に指定できるのでオススメです。
  return pd.merge(tmp_now, tmp_before, on = ["shop_id", "item_id"], how="left")

# ラグ特徴量作成開始
tmp = pd.DataFrame()
for i in range(1,max(df_3["date_block_num"] + 1)):
  tmp_now = df_3.filter(["date_block_num", "shop_id", "item_id"])\
                .query("date_block_num ==  @i")

  # 過去3ヶ月間のラグ特徴量
  for j in range(1, 4):
    tmp_now = func_lag("item_price", j)
    tmp_now = func_lag("category_cnt_month", j)
    tmp_now = func_lag("category_price", j)
    tmp_now = func_lag("category_sales", j)
    tmp_now = func_lag("shop_item_sales", j)
    tmp_now = func_lag("shop_sales", j)

  # 過去6ヶ月間のラグ特徴量
  for j in range(1, 7):
    tmp_now = func_lag("item_cnt_month", j)

  # 各月ごとのデータを縦結合して1つにまとめる
  tmp = pd.concat(
    [tmp, tmp_now]
    ,ignore_index = True
    )
  del tmp_now

# 以下の処理をまとめて実行
# ・ラグ特徴量を結合
# ・リークしている特徴量削除
# ・値段の変化率の特徴量作成
# ・欠損値を0に置換
# ・小数点第二位を四捨五入
data = round(
    pd.merge(df_3, tmp, on = ["date_block_num", "shop_id", "item_id"])\
      .drop(columns=["category_cnt_month", "category_sales", "shop_item_sales", "shop_sales"])\
      .assign(item_price_rate = lambda x:(x["item_price"] - x["item_price_1"]) / x["item_price_1"] * 100)\
      .fillna(0)
      ,1)

del tmp, i

データ加工はこれにて以上になります。 salesなどの4つの特徴量を削除しておりますが、これはリークと言う現象を防ぐためです。これらは目的変数であるitem_cnt_monthが分かる前提でできた特徴量のため、本来なら知りえない情報から不正に情報を得てしまうリスクがあります。ただ、ラグ特徴量の場合は過去の情報のため、今回は学習に使用しても問題ありません。

5.機械学習とモデル評価¶

いよいよ機械学習をやっていきます。
今回はLightGBMというGBDTアルゴリズムで機械学習をしていきます。
モデル評価には回帰分析で一般的な平均絶対値誤差(MAE)を使います。
また、学習データは交差検証（クロスバリデーション）という方法で分析をしていきます。
交差検証とは学習データをさらに複数のデータ(fold)に分割して評価を行う手法です。
時系列データの交差検証は以下のようなイメージで行います。

それでは、早速LightGBMのクラスを作っていきましょう。

In [9]:
class lgb_model:
  def __init__(self):
    # モデルと結果を保存するための変数を用意する
    self.model = None
    self.results_dict = {}
    self.va_scores = []

  def trains(self, num_round, va_period_list):
    for va_period in va_period_list:

      # 学習データを訓練データと検証データに分割する。
      is_tr = "date_block_num < @va_period"
      is_va = "date_block_num == @va_period "  #テストデータに合わせて検証データも1ヶ月分にする。
      tr_x = train.query(is_tr).drop(columns=[target])
      tr_y = train.query(is_tr).filter([target])
      va_x = train.query(is_va).drop(columns=[target])
      va_y = train.query(is_va).filter([target])

      # 特徴量と目的変数をLightGBMのデータ構造に変換する
      lgb_train = lgb.Dataset(tr_x, tr_y)
      lgb_eval = lgb.Dataset(va_x, va_y)
      num_round = num_round  # 学習回数

      # 学習の実行
      # 検証データもモデルに渡し、学習の進行とともにスコアがどう変わるかモニタリングする
      self.model = lgb.train(
          params
          ,lgb_train
          ,num_boost_round = num_round
          ,valid_names = ["train", "valid"]
          ,valid_sets = [lgb_train, lgb_eval]
          ,evals_result = self.results_dict
        )
      
      # バリデーションデータでのスコアの確認
      va_pred = self.model.predict(va_x)
      va_score = mean_absolute_error(va_y, va_pred)
      self.va_scores.append(va_score)
      
  def predict(self, predict_data):
      return np.round(self.model.predict(predict_data))   # 小数点第一位を四捨五入

LightGBMのクラスが出来上がりました。
このクラスを使って、いよいよ機械学習を始めます。
まずは、学習データを25ヵ月分、29ヵ月分、33ヵ月分で交差検証をしていきます。
